# 워커 모바일 앱 — QA 테스트 지침서

> 개발 지식 없이 따라할 수 있도록 작성된 단계별 테스트 지침서입니다.
> **✅ 통과** 항목은 이렇게 동작해야 정상, **❌ 실패** 항목은 이렇게 동작하면 버그입니다.

---

## 준비 사항

| 항목 | 내용 |
|------|------|
| 접속 URL | 개발 서버 주소 (팀 내 공유) |
| 권장 환경 | Chrome 모바일 에뮬레이션 (F12 → 기기 툴바 → iPhone 12 또는 Galaxy S20) |
| 실제 기기 권장 | SMS 인증 테스트 시 Android 또는 iOS 실기기 필요 |

## 시나리오 1 — 자동 로그인

### A. 자동 로그인 체크 후 로그인

1. 로그인 화면 진입 → **"자동 로그인" 체크박스가 표시되는지** 확인
2. 처음 방문 시 전화번호·비밀번호 칸이 비어있는지 확인
3. 체크박스에 체크 → 전화번호·비밀번호 입력 → **로그인 탭**
4. 로그인 성공 시 홈 화면으로 이동하는지 확인
5. F12 → Application → Local Storage 에서 `auto_login = true` 및 자격증명이 저장되어 있는지 확인
6. 하단 **로그아웃 탭**
7. 로그인 화면으로 이동하는지 확인 (토큰·자격증명은 유지된 상태)
8. 탭을 새로고침(또는 앱 재시작)
   - 세션이 유효하면 → ✅ 로그인 없이 홈으로 자동 이동
   - 세션이 만료되었으면 → 로그인 화면 표시 + 전화번호·비밀번호 자동 입력됨 확인

### B. 자동 로그인 미체크 후 로그인

1. 로그인 화면 진입 → 체크박스 **미체크** 상태로 로그인
2. 로그인 성공 후 F12 → Application → Local Storage 에서 `auto_login = false` 이고 자격증명이 없는지 확인
3. **로그아웃 탭** → F12 → Local Storage 가 비어있는지 확인
4. 탭 새로고침 → ✅ 자동 이동 없이 로그인 화면이 유지되는지 확인

> ❌ 미체크인데 자동으로 로그인되거나, 체크했는데 자동 입력이 안 되면 버그입니다.

## 시나리오 2 — 비밀번호 재설정

1. 로그인 화면에서 **"비밀번호 재설정"** 링크 탭
2. SMS 인증 화면(`/login/sms-verification`)으로 이동하는지 확인
3. 전화번호 입력
   - 11자리 미만 입력 시 → **"인증번호 받기" 버튼이 비활성** 상태인지 확인
   - 11자리 입력 완료 시 → 버튼 활성화 확인
4. **"인증번호 받기" 탭** → SMS 수신 대기
5. 수신한 6자리 코드 입력
   - 6자리 미만 시 → **"다음" 버튼이 비활성** 상태인지 확인
   - 6자리 입력 완료 시 → **"다음" 탭**
6. 비밀번호 설정 화면(`/login/set-password`)으로 이동하는지 확인
7. 새 비밀번호 입력
   - 조건(8자 이상, 영문, 숫자, 특수문자)이 실시간으로 표시되는지 확인
   - 조건 미충족 시 → 비밀번호 확인 필드가 비활성 상태인지 확인
   - 조건 모두 충족 시 → 비밀번호 확인 필드 활성화 및 자동 포커스 확인
8. 비밀번호 확인 입력
   - 불일치 시 → "비밀번호가 일치하지 않습니다" 문구 표시 확인
   - 일치 시 → **"다음" 탭**
9. ✅ "비밀번호가 변경되었습니다" 토스트 메시지 + 로그인 화면 이동 확인
10. ❌ API 실패 시 → 오류 토스트 표시 + 화면 유지 확인

---

## 시나리오 3 — 비밀번호 변경 (프로필)

1. 로그인 후 **프로필 화면**(`/profile`) 진입
2. **"비밀번호 변경"** 항목 탭
3. 비밀번호 변경 화면(`/profile/change-password`)으로 이동하는지 확인
4. **현재 비밀번호** 입력
5. **새 비밀번호** 입력
   - 조건(8자 이상, 영문, 숫자, 특수문자)이 실시간으로 표시되는지 확인
   - 조건 미충족 시 → **"변경" 버튼이 비활성** 상태인지 확인
6. 조건 모두 충족 후 **새 비밀번호 확인** 입력
   - 불일치 시 → "비밀번호가 일치하지 않습니다" 문구 표시 + 버튼 비활성 확인
   - 일치 시 → **"변경" 버튼 활성화** 확인
7. **"변경" 탭**
8. ✅ "비밀번호가 변경되었습니다" 토스트 + 프로필 화면(`/profile`)으로 이동 확인
9. ❌ 현재 비밀번호 오류 시 → 오류 토스트 + 화면 유지 확인

---

## 시나리오 4 — 패스워드 잠금

1. 로그인 화면에서 올바른 전화번호 + **잘못된 비밀번호** 입력 후 **"로그인" 탭**
2. 1~4회 실패 시 → 오류 토스트 표시 + 화면 유지 확인
3. **5회 실패** 시 → "5분 잠금" 안내 메시지가 표시되는지 확인
4. 잠금 중 로그인 재시도 → 잠금 안내 메시지가 다시 표시되는지 확인 (5분 내 로그인 불가)
5. 5분 대기 후 → ✅ 로그인 가능 상태로 복구 확인
6. 올바른 비밀번호 입력 → ✅ 로그인 성공 확인

---

## 시나리오 5 — 회원가입

### A. 타인 명의 SMS 인증 (주요 경로)

1. 로그인 화면 → **"회원 가입"** 탭
2. **"타인 명의 휴대폰이 있어요"** 선택
3. 전화번호 입력
   - 11자리 미만 → **"인증번호 받기" 비활성** 확인
   - 11자리 입력 → 버튼 활성화 → **"인증번호 받기" 탭**
4. 서버 응답 확인
   - **이미 가입된 번호** → ❌ 오류 토스트 표시 + 화면 유지 (가입 차단 확인)
   - SMS 발송 성공 → 6자리 코드 입력창 표시
5. 수신한 6자리 코드 입력
   - 6자리 미만 → **"인증하기" 비활성** 확인
   - 6자리 완성 → **"인증하기" 탭**
6. ✅ 인증 성공 → 약관 동의 화면으로 이동

### B. 내 명의 NICE 인증

1. **"내 명의 휴대폰이 있어요"** 선택 → NICE 외부 페이지로 리다이렉트
2. NICE 인증 완료 시 → ✅ 약관 동의 화면으로 이동
3. NICE 인증 실패/취소 시 → ❌ 오류 토스트 + 회원가입 화면 복귀

### C. 약관 동의 및 정보 입력

1. 약관 동의 화면에서 **"동의하기" 탭**
2. 내국인/외국인 선택

**내국인 (주민등록번호)**
- 이름 입력 (한글만, 최대 4자)
- 주민등록번호 앞 6자리 입력 → 자동으로 뒷자리 필드로 포커스 이동하는지 확인
- 뒷자리 7자리 입력

**외국인 (외국인등록번호)**
- 한글이름 입력 (최대 4자) + 영문이름 (선택)
- 외국인등록번호 앞 6자리 → 뒷자리 7자리 입력
- 여권번호로 가입하려면 **"여권번호로 가입하기" 탭** → 아래 여권 항목으로

**외국인 (여권번호)**
- 한글이름, 영문이름, 성별, 여권번호 (영문+숫자 최대 12자), 국적, 생년월일 입력

3. 주소 입력 → **카카오 주소 검색 팝업이 열리는지** 확인 → 주소 선택
4. **"다음" 탭** → 비밀번호 설정 화면 이동
5. 비밀번호 입력
   - 조건(8자 이상, 영문, 숫자, 특수문자) 실시간 표시 확인
   - 조건 미충족 또는 확인 불일치 시 → **"다음" 비활성** 확인
   - 모두 충족 시 → **"다음" 활성** → **"다음" 탭**
6. ✅ 가입 완료 화면 표시
7. ❌ API 실패 시 → 오류 토스트 + 화면 유지

## 시나리오 6 — 온보딩

> 주민등록번호 또는 외국인등록번호로 가입한 계정으로 최초 로그인 시 진행됩니다.
> 여권번호 가입 계정은 온보딩 없이 홈 화면으로 바로 이동합니다.

1. 최초 로그인 → 온보딩 시작 화면에서 **이름이 포함된 환영 메시지** 표시 확인
2. **"시작하기" 탭**
3. 근무자 유형 선택

**일반 근무자**
- 급여 지급 방식 화면에서 **"소속 회사" 옵션이 없는지** 확인

**용역 근무자**
- 용역회사 선택 화면 → 드롭다운 목록 로딩 확인
- 회사 선택 후 다음 → 급여 지급 방식 화면에서 **"소속 회사" 옵션이 있는지** 확인

**장비기사**
- 장비기사 정보 입력 → **"다음" 탭**
- ✅ 급여계좌·서류 화면을 건너뛰고 일급 입력 화면으로 바로 이동하는지 확인 → **7번으로 이동**

4. 급여 지급 방식 선택
   - **본인 계좌** → 은행·계좌번호·예금주명 입력
   - **가족 계좌** → 예금주명·관계·은행·계좌번호 입력
   - **소속 회사** → ✅ 일급 입력 화면으로 바로 이동 (7번으로 이동)
5. 저장 후 서류 등록 화면 표시
   - **"나중에 하기"** 탭 → ✅ 홈 이동 + 서류 미완료 배너 표시 확인
   - **"다음"** 탭 → 일급 입력 화면 이동

6. 일급 입력
   - 0 입력 시 → **저장 버튼 비활성** 확인
   - 금액 입력 후 → **"저장" 탭**
7. ✅ 완료 팝업 표시 → "확인" 탭 → 홈 화면 이동
8. ❌ 저장 도중 실패 시 → 오류 토스트 (서버에 일부 저장된 상태일 수 있으니 주의)

## 시나리오 7 — 계좌 등록

1. 홈 화면의 **"계좌 정보 입력이 완료되지 않았어요" 배너** 탭 또는 프로필 → 계좌정보 항목 탭
2. 급여 지급 방식 선택

**본인 계좌**
1. 은행 드롭다운 탭 → 목록이 열리는지 확인 → 은행 선택
2. 계좌번호 입력 (7자리 미만 시 저장 버튼 비활성 확인)
3. 예금주명 입력
4. **"저장" 탭**

**가족 계좌**
1. 예금주 이름 입력
2. 관계 드롭다운 → 선택
3. 은행 선택 → 계좌번호 입력
4. **"저장" 탭**

**소속 회사** (용역 근무자에게만 표시)
- 탭 즉시 API 호출

3. ✅ 저장 성공 → 프로필 화면 이동 + 홈의 계좌 배너가 사라지는지 확인
4. ❌ 실패 시 → 오류 토스트 표시

---

## 시나리오 8 — 서류 업로드

1. 프로필 → 서류 등록 화면 진입
2. 서류 타입별 UI 확인
   - **미제출 서류** → "파일 선택" + "사진 촬영" 버튼 표시 확인
   - **제출 완료 서류** → "보기" 버튼 표시 확인
   - **전자서명 서류** → 탭 시 "전자서명 서류는 별도 진행이 필요합니다" 토스트 표시 확인
   - **장비 면허** → 탭 시 장비 화면으로 이동 확인

3. **"파일 선택" 탭** → 파일 선택 다이얼로그 열림 확인
4. 파일 선택
   - **10MB 초과 파일** → ❌ 업로드 전 오류 토스트 표시
   - **잘못된 형식** → ❌ 오류 토스트 표시
   - **정상 파일** → 업로드 중 스피너 표시 + 다른 서류 버튼 비활성 확인
5. ✅ 업로드 성공 → "제출되었습니다" 토스트 + 해당 서류가 "보기" 버튼으로 변경 확인
6. ❌ 업로드 실패 → 오류 토스트 + 버튼 다시 활성화 확인

---

## 시나리오 9 — 계약서

1. 하단 탭 → **"계약서"** 진입
2. 미서명 계약이 있을 경우 → 상단에 **"서명하지 않은 근로계약서가 있어요"** 경고 배너 표시 확인
3. 연도 드롭다운 확인
   - 기본값이 현재 연도인지 확인
   - 탭 시 드롭다운 열림 + 최근 3개년 표시 확인
   - 다른 연도 선택 시 → 해당 연도 계약서 목록으로 갱신 확인

4. 월별 계약서 카드에서 서명 상태별 UI 확인

| 상태 | 표시 내용 |
|------|-----------|
| 서명 필요 (AWAITING_WORKER) | 빨간 테두리 강조 + "서명 필요" 뱃지 + **서명하기** 버튼 |
| 관리자 승인 대기 | "관리자 승인 대기" 뱃지, 버튼 없음 |
| 완료 (COMPLETED) | "완료" 뱃지 + **열람** 버튼 |
| 승인 대기 (SENT) | "승인 대기" 뱃지 + **열람** 버튼 |
| 미발송 (DRAFT) | "미발송" 뱃지, 버튼 없음 |
| 반려 (REJECTED) | "반려" 뱃지, 버튼 없음 |

5. **"서명하기" 탭** → 새 탭에서 eformsign 서명 페이지 열림 확인 (탭 중 스피너 표시)
   - ✅ 서명 페이지 정상 열림
   - ❌ 실패 시 오류 토스트 + 새 탭 닫힘 확인

6. **"열람" 탭** → 새 탭에서 PDF 열림 확인
   - ✅ PDF 정상 표시
   - ❌ 실패 시 오류 토스트 + 새 탭 닫힘 확인

7. 계약서 로드 실패 시 → 재시도 버튼 표시 확인
8. 해당 연도 계약서 없을 때 → "{연도}년 계약서가 없습니다" 문구 표시 확인

## 시나리오 10 — 출퇴근

### 출근

1. 홈 화면 확인
   - 당일 출근 2회 완료 상태 → **"출근하기" 버튼 비활성 + 툴팁** 표시 확인
   - 0~1회 출근 상태 → **"출근하기" 버튼 활성** 확인
2. **"출근하기" 탭** → QR 스캐너 열림 확인
3. 현장 QR 코드 스캔
4. GPS 권한 확인
   - 권한 미허용 상태 → 위치 권한 요청 팝업 표시 확인
   - 허용 또는 거부 모두 → 출근 진행 (GPS는 선택사항)
5. 출근 API 호출 중 → 버튼 연속 탭이 차단되는지 확인
6. ✅ 출근 성공 → "출근 완료 [현장명]" 토스트 + 현장 체크인 상태 표시 확인
7. ❌ 실패 시 → 오류 토스트 + 재시도 가능 확인

### 퇴근

1. 출근 완료 상태에서 **"퇴근하기" 탭**
2. 퇴근 확인 다이얼로그 표시 확인
   - **"취소"** → 다이얼로그 닫힘 + 근무중 상태 유지 확인
   - **"확인"** → GPS 요청 후 퇴근 API 호출
3. ✅ 퇴근 성공 → 오늘의 근무 기록 표시 확인
4. ❌ 실패 시 → 오류 토스트 표시

## 시나리오 11 — 출퇴근 기록 조회

### 달력 뷰

1. 하단 탭 → **"출퇴근"** 진입 → 달력 뷰 기본 표시 확인
2. 이전/다음 월 화살표 탭 → 월 이동 확인
3. 출근 기록 있는 날짜에 컬러 점 마커 표시 확인
4. **날짜 탭** → 해당 날짜 상세 화면으로 이동 확인
5. 현장 드롭다운 확인
   - 특정 현장 선택 → 해당 현장 날짜만 달력에 표시되는지 확인
   - 전체 선택 → 모든 현장 마커 표시 확인
6. 이번 달 요약 카드 확인
   - 총 출역일, 총 공수 (현장별 색상 구분), 예상 노임 표시 확인

### 목록 뷰

1. **"목록"** 탭 → 목록 뷰로 전환 확인
2. 요약 카드 수치가 달력 뷰와 동일한지 확인
3. 현장 드롭다운 (오늘 + 과거 현장 모두 포함)
   - 특정 현장 선택 → 해당 현장 카드만 표시 확인
4. 일별 그룹 카드에서 현장명, 출퇴근 시간, 공수, 예상 노임 표시 확인
5. 오늘 기록 카드에서 정정 요청 버튼 표시 여부 확인
   - 정정 가능 → **"정정 요청" 버튼 활성** + 탭 시 정정 다이얼로그 표시
   - 이미 요청됨 → **"정정 요청" 버튼 비활성** 확인
   - 과거 날짜 → 정정 요청 버튼 없음 확인

### 상세 화면

1. 달력에서 날짜 탭 → 상세 화면 이동 확인
2. 이전 날 이동 화살표 확인 (미래 날짜 버튼은 표시되지 않음)
3. 출근 기록 있을 때 → 현장명, 시간, 공수 카드 표시 확인
4. 출근 기록 없을 때 → 빈 상태 안내 문구 표시 확인
5. 당일 상세에서는 정정 요청 버튼 + 안내 배너 표시 확인
6. 과거 날짜 상세에서는 정정 요청 버튼 없음 확인

## 시나리오 12 — 야근 신청

1. 출근 완료 상태 홈 화면에서 **야근 중 뱃지** 표시 여부 확인
   - 정규 시간 이내 → 야근 뱃지 없음
   - 정규 시간 초과 → **"야근 중"** 뱃지 표시 확인
2. **"야근 신청" 버튼 탭** → 야근 신청 확인 다이얼로그 표시 확인
3. 다이얼로그 선택
   - **"취소"** → 다이얼로그 닫힘 + 야근 상태 유지 확인
   - **"확인"** → 야근 신청 API 호출
4. ✅ 신청 성공 → 야근 신청 완료 토스트 + 신청 상태 변경 확인
5. ❌ 실패 시 → 오류 토스트 + 재시도 가능 확인
6. (관리자 처리 후 확인)
   - 승인 → ✅ 야근 시간이 급여에 반영됨 확인
   - 반려 → 반려 안내 확인

## 시나리오 13 — 출퇴근 정정요청

1. 홈 화면 또는 출퇴근 기록 화면에서 **"정정 요청" 버튼 탭**
2. 정정 요청 유형 선택
   - 출근 시간 정정 → 수정할 출근 시간 입력 + 사유 입력
   - 퇴근 시간 정정 → 수정할 퇴근 시간 입력 + 사유 입력
   - 출근 기록 없음 → 출근 누락 사유 입력
   - 퇴근 기록 없음 → 퇴근 누락 사유 입력
3. 사유가 비어있을 때 → **"제출" 버튼 비활성** 확인
4. 사유 입력 후 → **"제출" 탭**
5. ✅ 성공 → "정정요청 완료" 토스트 + 기록 상태 변경 확인
6. ❌ 실패 시 → 오류 토스트 + 화면 유지 + 재시도 가능 확인
7. (관리자 처리 후 확인)
   - 승인 → ✅ 정정된 시간으로 기록 업데이트 확인
   - 반려 → 원본 기록 유지 확인

## 시나리오 15 — 엣지 케이스

### A. 탭 닫기 후 재진입

1. SMS 인증 완료 후 주민번호 입력 화면에서 **탭을 닫고 다시 URL 접속**
2. ✅ 처음(로그인 화면)부터 다시 시작되면 정상
3. ❌ 입력했던 정보가 유지되면 버그 보고

### B. 이름 5자 입력

1. 회원가입 또는 프로필 이름 입력 필드에 **한글 5자 입력 시도**
2. ✅ 4자로 잘리고 오류 문구 표시되면 정상
3. ❌ 5자가 그대로 입력되면 버그 보고

### C. 네트워크 오프라인

1. F12 → Network 탭 → **"Offline"** 설정
2. **"인증번호 받기" 탭**
3. ✅ 오류 토스트가 표시되면 정상
4. ❌ 흰 화면이 나타나거나 아무 반응 없으면 버그 보고

### D. 온보딩 뒤로가기

1. 온보딩 진행 중 **뒤로가기 탭**
2. ✅ "나가기" 확인 다이얼로그 표시 확인
3. **"나가기"** 선택 → ✅ 로그아웃 후 로그인 화면으로 이동 확인
4. **"계속하기"** 선택 → ✅ 온보딩 화면 유지 확인

### E. 비밀번호 64자 초과

1. 비밀번호 입력 필드에 **65자 이상 입력**
2. ✅ "패스워드는 최대 64자까지 설정할 수 있습니다" 오류 문구 표시되면 정상
3. ❌ 오류 문구 없이 입력되면 버그 보고
